git版本控制的配置

发表于 2016-12-10 | 分类于 git | | 阅读次数

这篇文章主要讲下Mac上git版本控制的使用，及遇到的坑

一、git的配置

  • 首先找到代码托管服务，github或者码云(推荐 国内速度快 有1G的免费私有空间)
  • 公司开发都会选择私有库，创建项目，创建成功后有一个默认的master分支
  • 将master分支设置为保护分支，点击branches，新建分支develop为公共开发分支，新建分支develop_name为每个开发人员的个人分支
  • 然后git clone项目，git branch查看当前分支为master
  • 输入git fetch origin develop:develop从远程拉取develop分支，以同样方式拉取个人分支
  • git checkout develop切换到develop分支，git branch查看当前分支颜色为高亮

一、git的常用命令

  • git add . //将改变的代码添加到git
  • git commit -m "备注信息" //备注提交信息
  • git git push origin master 提交到远程仓库
  • 设置用户名$ git config --global user.name "你的昵称"
  • 设置用户邮箱$ git config --global user.email "你的邮箱地址"
  • 查看git设置列表信息$ git config --list
  • 查看用户名$ git config user.name
  • 生成ssh秘钥$ ssh-keygen -t rsa -C “your@email.com”
  • 进入ssh秘钥目录下$ ~/.ssh,目录下的公共秘钥文件id_rsa.pub

二、同一台设备上给两个github账号配置ssh key

  • 现在我有两个git hub账号，但是一个ssh key只能用于一个git hub账号，所以就需要下面的方法来生成不同的ssh key：
  • 首先用通常方法生成一个默认的ssh key
  • 然后，ssh-keygen -t rsa -C "第二个github邮箱"
  • 回车，到这一步输入id_rsa需要保存的文件名，如下
  • Enter file in which to save the key (/Users/.ssh/id_rsa): github_bolg_id_rsa，至此生成第二个可用的ssh key
  • 默认只读取id_rsa，为了让SSH识别新的私钥，需将其添加到SSH agent中：
    ssh-add ~/.ssh/github_bolg_id_rsa
  • 在ssh目录下创建一个config文件，vi config将下面的内容拷贝到config文件中，将第二个host后的名字改为你想配置的名字，IdentityFile设置为你生成第二个ssh key的路径
  • 然后将连接到github的链接改为git@此处为在config文件中配置的账户名（我的是github_blog）:rocksky/rocksky.github.io.git，此时就可以用新建的ssh key的秘钥了
