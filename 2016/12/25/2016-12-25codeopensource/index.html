过程中遇到的坑

  • 验证没有通过，各种尝试后还报下面的错，接近奔溃的边缘，只留了一丝希望
  • 可以看出上面一直报source_files的错说明.podspec文件的source_files配置一直是不对的，注意看这里s.source_files = 'skydatemaster/**/*.{h,m}'// skydatemaster的名字必须是你开源代码存放的文件夹名 而且这个文件件要与. podspec文件在同一级文件目录下，这是个好大的坑，坑到你心理默念“放弃吧！放弃吧！。。。。”

更新cocoapods的开源库

  • 配置github正确的ssh
  • git clone 克隆github上开源的项目
  • 修改代码和SKYTools.podspec配置文件（版本号）
  • 将修改的代码更新到github的远程仓库
  • 添加版本号git tag -a 1.0.1 -m"version 1.0.1"
  • 查看tag是否添加成功git tag
  • 将本地的添加的tag同步到远程仓库git push --tags
  • 检查pod trunk 信息pod trunk me
  • 验证项目pod spec lint SKYTools.podspec --verbose --use-libraries
  • 将项目push到cocoapods上pod trunk push SKYTools.podspec
  • 提交成功后搜索pod search skytools发现没有搜索到最新的版本，执行rm ~/Library/Caches/CocoaPods/search_index.json重新pod search 开源项目
  • 如果还是搜索不到就执行更新pod库:pod setup
