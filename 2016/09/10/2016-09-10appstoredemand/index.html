手机尺寸

  • 屏幕的大小
    iphone 4/4s 尺寸:3.5英寸 分辨率:640x960 屏幕上的实际的点:320x480
    iphone 5/5s 尺寸: 4.0 分辨率: 640x1136 实际的点：320x568
    iphone6 尺寸:4.7 分辨率:750x1334 实际的点:(375x667)
    iphone 6Plus 尺寸:5.5 分辨率:1080x1920 实际:(414x736)

填写AppStore信息

  • 首先需要一个99$的开发者账号，类型可以是公司的或个人的
  • 证书配置：需要两个根证书，有推送的话还需要两个推送证书，三个配置文件（一个测试，一个发布，一个商店），详细配置请看这里
  • 截图5.5英寸 5张 大logo 1024x1024 一张
  • 类别(例如 工具 商务)
  • 描述
  • 关键词
  • 版权
  • 商务代表联系人信息 名字 地址 电话 邮箱
  • 技术支持网址
  • 营销网址
  • 审核联系人信息
  • 演示账户

App 启动图–Logo

  • 启动图size 只适配iPhone 竖屏
    640x1136 （Retina 4）
    750x1334 （4.7）
    1242x2208（5.5）

  • logo尺寸 只适配iPhone
    40x40
    58x58
    60x60
    80x80
    87x87
    120x120
    180x180

  • 提交appstore时，查看是否使用了广告标识符
    1、打开终端cd到要检查的文件的根目录。
    2、执行下列语句：grep -r advertisingIdentifier . （别少了最后那个点号）
    3、若包含，则会输出Binary …..的字眼，勾选广告标识

自己服务器维护ipa更新

参考:http://www.cnblogs.com/xiaoc1314/p/5952555.html

  1. iOS端提供ipa安装包和plist文件
  2. 服务器端需配置https环境,及关联plist和ipa包
  3. 服务器端需提供plist文件下载地址和图片(57x57.png 和512x512.jpg 共计两张图片地址)下载地址
  4. 所有地址必须是https的
  5. 服务器端配置请参考此处
  • {
    appUrl = “https://www.pgyer.com/yaolaiyuntest“;
    build = 4;
    downloadURL = “itms-services://?action=download-manifest&url=https://www.pgyer.com/app/plist/05c19907143ff564a87d922f7ec5894b“;
    lastBuild = 4;
    releaseNote = “”;
    versionCode = 1;
    versionName = “1.0.1”;
    }

App 发布

  • 手动发布时候居然碰到“no suitable application records were found,Verify your bundle identifier is correct.”问题。 网上查了一下原来是，我少操作了一步，￼
    在网页上面还有个按钮“Ready for upload”, 点了以后状态就成了“Waiting for upload” 这个时候才能上传！唉….必须记录一下!

App 异常 系统Bug

  1. 下载App后无法使用网络,蜂窝数据中查看不到该APP(iOS10),解决办法
  2. 下载App后,没有网络,无法使用App,蜂窝数据中没有此App,(iOS8),解决办法 关闭wifi重新打开App即可

App 被拒原因汇总

  1. 商店资料描述中含有链接,指向公司网站
  2. 没有注册选项时要告诉苹果用户获取账号密码的方式 app的使用对象
  3. 所有测试数据不能含有测试字样
  4. App没兼容最新iOS系统
  5. 使用idfa广告标识要再提交时选择使用idfa
  6. 绕过苹果的付费渠道，我们之前游戏里的用兑换码兑换金币
